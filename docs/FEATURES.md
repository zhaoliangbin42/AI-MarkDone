# AI-MarkDone — Product Capabilities & Acceptance (Authoritative)

目的：用 **一个** 权威文档固化当前产品能力、验收口径与测试映射，避免多份功能文档漂移。

范围：本文件定义“能力在支持的平台上应如何工作”，并持有当前平台支持状态与支持级别。当前落地运行时边界同步记录在 `docs/architecture/CURRENT_STATE.md`。

术语：

- **Driver**：站点适配 + DOM 采集/注入 + browser APIs（不得依赖 UI/Service）
- **Service**：分为 pure/domain service 与 content-facing feature service（都不得依赖 UI；前者避免直接触碰 DOM/browser APIs，后者允许处理 DOM clone / parser node / content fragment）
- **UI**：Surface 渲染与交互，默认使用 Shadow DOM；catalog 中明确登记的 light-DOM host 与 extension page 是受控例外（不得直写存储；不得包含平台差异选择器）

## A) Product Decisions（产品决策）

这些决策属于当前稳定产品行为。改动必须更新本文件并补齐回归证据。

| Topic | Decision | Rationale |
|---|---|---|
| Platform support policy | Capability behavior and support level are defined here; implementation inventory lives in `CURRENT_STATE.md` | 把“产品能力/支持承诺”与“当前实现路径”分开治理，减少文档职责混叠。 |
| UI entrypoints | **No legacy global toolbar**; compact global page controls are allowed only for ChatGPT navigation, bookmarks, Prompts, and Detached Reader | 减少注入点与 UI 干扰；常规消息动作仍聚焦“每条消息工具栏 + ReaderPanel”，但右下角 ChatGPT message stepper 所在的页面级控制组可承载低频全局动作。ChatGPT 书签面板入口、Detached Reader 和 Prompts 不属于单条消息 toolbar 动作，应放在同组全局按钮中。 |
| Action icon click | Background sends `ping` → `ui:toggle_toolbar` → Content toggles **BookmarksPanel** | 扩展图标作为稳定入口；点击前先轻量确认当前 ChatGPT tab 的 content script 可达，失效 tab 静默降级，页面恢复后通过 `content:ready` 重新接回。 |
| Per-message toolbar placement | Official action bar row; verified Deep Research report-stack anchor only | 普通消息仍只进入官方 action row，缺失时不做通用正文 fallback；唯一例外是已验证的 `iframe[title="internal://deep-research"]`，它必须位于 ChatGPT 的 `data-conversation-screenshot-content` 报告根内，工具栏才在报告内容栈底部右对齐。未知或无锚点 iframe fail closed。 |
| Injection algorithm | MO signal classification + debounced incremental reconcile + ChatGPT message lifecycle + targeted stale recovery + stable-parent root observation | SPA/React 更稳定；消息内文本/子树 mutation 只推进所属消息，无关文本不触发扫描；message 可先出现、官方 action bar 后补出现，可归属的 action-row hydration 只推进对应消息状态。ChatGPT 直接替换 conversation `main` 时，由稳定父级 observer 触发一次 full rebuild 并继续监听；官网单独移除 toolbar host 时定向重建，自身 intentional removal 不反弹。若某条消息在 hydration / network jitter 窗口进入 `anchor_pending` 或 `stale`，只对该消息做递增退避的 incremental reconcile。整页重扫仅保留给 init、route/message structure/root replacement 或无法归属的异常结构；scheduled reconcile 后不得再全量遍历一次。每条工具栏仍立即提供全部正式动作，但 Copy PNG 的任务进度子树/CSS 只在任务真实开始时创建，避免长对话为未使用的低频反馈预付 DOM 成本。 |
| Heavy feature loading | Reader, Bookmarks, Save dialogs, Copy PNG, and formula assets load only on first real use | 页面启动只加载工具栏、目录、路由与轻量 controller/port；打开 Reader/Bookmarks、保存消息/书签、执行 Copy PNG/公式资产动作或首次公式 SVG preview 时，才经同一个 lazy content-feature façade 从扩展自身 origin 加载对应 ES module chunks。模块失败可在下一次用户动作重试；不得从 ChatGPT origin 请求 chunk，也不得因 lazy loading 改变按钮、面板或导出语义。 |
| UI Surface system | One appearance value, explicit appearance scopes, shared Surface lifecycle, family-owned chrome, and executable coverage | 页面内 UI 统一接收不可变 `AppearanceSnapshot`，并通过 page、ShadowRoot 或已登记的 light-DOM `AppearanceScope` 应用 token；`SurfaceSession` 为 panel/modal/anchored/inline 提供 focus、dismiss、position、motion 和 teardown 合同，feature controller 只保留业务编排与 adapter-owned geometry。每个用户可见 Surface 都必须在 design catalog 与 test-only coverage manifest 中登记真实入口、owner、responsive contract、Chrome/Firefox 目标、trigger test 和真实组件 fixture。该收敛不改变用户 workflow、设置 schema、runtime protocol 或 heavy-feature lazy boundary。 |
| Image export architecture | Three user entries, two semantic inputs, one scheduler and one primary export renderer | 当前消息 Copy PNG、Save Messages PNG 与单公式 PNG/SVG/MathML 保留原入口和交付行为。消息与 authoritative TeX 资产进入按需创建的 extension-origin `export-renderer.html`；无法传输 DOM 的 `dom-only` PNG 是 `renderFormulaAsset()` 背后的唯一 content-side compatibility exception，但仍与 host job 共用同一个高内存 FIFO scheduler。消息入口只提交 `ExportDocumentV1`，可信公式只提交结构化 spec；调用方不得提交 HTML/CSS/renderer function。Renderer 不读 storage、不联网、不写剪贴板、不触发下载，交付副作用继续由 content driver 持有。 |
| Formula source output | Click-copy and Markdown source copy/export use the same 5-format model, with separate settings for single-formula clicks and whole-message Markdown exits | 保持 Reader/renderer 内部 Markdown canonical，只在剪贴板或 Markdown 文件写出前转换公式 wrapper；用户可以分别控制“点单个公式复制”和“复制/保存整段 Markdown 源码”的输出习惯。 |
| ChatGPT message navigation | Default-on AI-MarkDone directory rail + optional lower-right controls | AI-MarkDone 右侧目录条是默认开启、用户可关闭的能力，由 `chatgptDirectory.enabled` 控制；启用 AI-MarkDone 目录条时同步隐藏可确认的 ChatGPT 官方对话导航，找不到候选则 no-op。右下角轻量 page-control cluster 由 `ChatGPTMessageStepperController` 独立持有，默认显示书签面板入口、页面收藏、Detached Reader Split View、Prompts、上一条/下一条入口；书签面板入口使用 AI-MarkDone 品牌 Logo、固定贴底，替代 ChatGPT header 入口并常驻在 ChatGPT 页面右下角，`chatgptBehavior.showPageBookmarkControl`、`chatgptBehavior.showDetachedReaderControl`、`chatgptBehavior.showPromptControl` 与 `chatgptBehavior.showMessageStepper` 分别控制其余按钮，其中 `showMessageStepper` 仅控制 Previous/Next。左右方向键导航由 `chatgptBehavior.enableArrowKeyMessageNavigation` 单独启用/关闭；ChatGPT 对话与 composer 输入条宽度通过 `chatgptBehavior.pageWidthScale` 在 100%–200% 内同步调大，100% 表示不改官方页面宽度。Reader locate、Bookmark Go 与 pending navigation 继续复用同一条 ChatGPT same-page navigation helper。 |
| ChatGPT send position restore | Optional, default-on page behavior under `chatgptBehavior.restorePositionAfterSend` | 发送后恢复阅读位置默认开启，只在用户主动发送时短生命周期 arm：记录主滚动容器和顶部附近 turn anchor，ChatGPT 跳底或水合位移后尽快 instant restore；用户可在 ChatGPT Settings 关闭；当有效的 `chatgptBehavior.inputEnhancement.enterKeyNewline` 开启时，官方输入框普通 Enter 被视为换行而非发送，Cmd/Ctrl + Enter 仍按发送处理；用户手动滚动、触摸、指针/键盘导航、官方滚到底部或显式 Reader/Bookmark 导航会释放，不进入 Reader/Export/Copy/Bookmark 正文链路。 |
| ChatGPT Input Enhancement | Default-on, independently configurable visible-marker enhancement of the official composer | 新安装的入口可用性与运行总开关均默认开启；升级用户按旧 Markdown / Enter 设置无损迁移。Settings 中的 `available` 只控制入口是否出现，composer 弹层中的 `enabled` 控制运行与按钮高亮；两者关闭均保留 Enter、粗体、列表和公式子项。该能力只通过原生范围编辑增强官方 ProseMirror/textarea，不建立富文本模型、不重写宿主 DOM。按钮 host 与官方加号动作容器并列，并使用 AI-MarkDone tooltip。Cmd/Ctrl + B 写入或移除可见 `**`；开启对应列表类型后，普通 Enter 续写、拆分或退出列表，有序列表保持 CommonMark-aware 同级连续编号和两阶段 Backspace 语义。行内 `$...$` 与行间 `$$...$$` 公式可分别使用轻量 MathJax 浮层预览和仅在公式环境内出现的 `\\` 片段联想；公式不在输入框内渲染，不支持 LaTeX Workshop `@` 快捷语法。Cmd/Ctrl + Enter 发送；Shift + Enter、IME、代码围栏和不可安全确认的结构交还宿主；Reader 输入框接入不属于当前版本。 |

#### Copy goldens（回归策略）

- Goldens：`tests/fixtures/expected/copy/chatgpt/<fixture>.md`
- Parity tests：从 ChatGPT mocks 读取 fixture，跑 rewrite pipeline，对比 golden
- Golden 更新脚本：`scripts/update-copy-goldens.ts`（ChatGPT-only，用于“有意识更新 baseline”，不进入默认 test 流）

#### LaTeX click safety guard（必须落实，避免破坏宿主页面）

当 LaTeX click 默认开启或被用户重新开启时，必须满足：

- selection 非空（用户正在选择文本）时 **不触发复制**，不拦截事件链。
- 仅在确认要复制时才 `preventDefault/stopPropagation`（尽量不干扰宿主交互）。
- hover 公式时可以展示 Settings 允许的单公式 PNG/SVG 复制与保存动作；这些动作必须继续消费同一份平台公式提取结果及其 source confidence，不得增加第二套公式源码解析链路或把 heuristic 文本升级成 authoritative TeX。

### D.2 手工验收清单（在支持的平台上）

- 刷新 / 切换对话 / 连续生成：每条已出现官方 action bar 的 assistant 消息最终出现工具栏（不重复、不漂移）；ChatGPT 官网中 action row 后 hydrate 时应局部推进对应消息状态，而不是常规触发整页补扫

- Reader：打开/翻页/复制，关闭无残留
- LaTeX click：可用且不影响用户选择/复制公式文本（selection guard 生效）
