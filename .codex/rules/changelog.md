# CHANGELOG

`CHANGELOG.md` is user-facing. Write what changed for users, not how the code changed internally.

## Include

- New user-visible capabilities.
- User-visible behavior changes.
- Bug fixes with a product-facing description.
- Meaningful performance or reliability improvements when users benefit directly.

## Exclude

- File paths.
- Function, class, or method names.
- Pure refactors, tests, or documentation work.
- Internal implementation details.

## Format

- Keep entries in English.
- Use standard categories such as `Added`, `Changed`, and `Fixed`.
- Prefer concise benefit-oriented phrasing.

## Example

```md
### Fixed
- Reader panel navigation no longer loses the current item after a page rerender.
```
