# 更新日志

# 4.2.0
2026-04-26

看了网友的反馈，我发现 Gemini、DeepSeek 和 Claude 平台其实都已经存在一些适配问题。结合我现有的精力想了想，与其做大而全，不如好好把精力聚焦在 ChatGPT 这一个平台上。实际上，ChatGPT 上已经有足够多的效率痛点需要解决，并且这也是我平时使用的主力平台，因此我的体会也是最深的。

实践出真知。我认为只有当我深入体会之后，才能更有针对性地做出符合真实场景的提效功能。并且，现有市场上也有一些项目兼顾了这些需求。再三考虑后，我决定接下来会逐步放弃 Gemini、DeepSeek 和 Claude 平台的开发，专注于 ChatGPT，回到最开始的目标线上。

当然，这几个平台也有替代插件，欢迎大家下载使用。

对于 Gemini，目前做得最好的插件应该是 [Gemini Voyager](https://github.com/Nagi-ovo/gemini-voyager)。

对于全平台书签功能，我之前用过一个非常喜欢的插件是 [Timeline](https://github.com/houyanchao/Timeline)。

我相信这两个插件在另外几个平台中应该能满足大家绝大多数的需求。当然，我也会在 ChatGPT 中沿着最初的宗旨继续前进，在保持克制、不打扰的前提下，开发出一个从理工科研究生角度来说真正趁手的 ChatGPT 插件，帮助大家提高使用效率，补齐 ChatGPT 体验上最后一公里的短板。

这次更新还增加了两个非常实用的功能，分别是导出消息为图片，以及导航栏的展开模式。其中，导出消息为图片是我个人比较喜欢的功能。平时，ChatGPT 消息分享主要还是以链接分享或者截图分享为主，但是对于很多内容比较长的对话，就需要用到长截图，上手难度会大幅增加。因此，在充分考虑用户分享场景后，我给插件添加了一键复制消息为图片以及批量导出消息为图片这两个功能。

另外，之前导航栏调的手风琴效果，好看是好看，但实用性欠佳。因为如果窗口中有几十条消息，我们无法一眼定位到想要的那条消息，而是要来回滑动去寻找。因此这个版本中引入了导航栏的全展开模式，方便用户一眼定位消息。

## 新增
- 新增对 Safari 浏览器和 Firefox 浏览器的支持。
- 增加消息批量渲染成图片功能（感谢小红书用户 @小锋iDyll）。
- 新增消息直接复制成图片功能，适合快速将消息分享给朋友（感谢小红书用户 @小锋iDyll）。
- 消息渲染成图片的宽度、清晰度可以在设置中进行修改，以适应目标分享平台的最佳视觉宽度。
- 新增导航栏的展开模式，方便一次性看清所有消息的内容摘要。
- 新增导航栏的开关，方便用户关闭导航栏。

## 修复
- 修复设置面板中的布局问题。
- 修复书签文件命名的 invalid 提示不够明确的问题（感谢小红书用户 @Colin的AI杠杆）。
- 修复导航栏点击一次无法正确定位到目标消息的问题（感谢小红书用户 @谦卑于世）。

## 原理解析
- 之前导航栏点击一次之后，会跳转到对应消息所在的位置。但是这个时候会触发 ChatGPT 官方的增量加载，而加载出来的内容会导致布局有所偏移，进而导致定位也发生偏差。所以之前导航栏需要点击第一次并跳转到对应位置之后，等网页稳定下来，再点击一次才能到达目标位置。现在的方案是点击一次之后，会先尝试跳转到对应的位置，然后监测增量加载之后的坐标变化，再进行一次跳转。第二次跳转对用户来说几乎无感。

# 4.1.2
2026-04-22

2026年4月22日凌晨，ChatGPT进行了一次较大的版本更新，导致页面内信息的全量加载变成了增量加载，原有的信息识别算法全部失效，影响范围包括但不限于：折叠条功能直接失效、阅读器只能识别到最终渲染的4~5条消息等。

鉴于最初开发折叠条的目的是为了提高页面的加载速度，降低渲染压力，如今 ChatGPT 已经官方支持增量加载，会自动卸载非活跃节点，性能得到大幅提升。另外也由于 ChatGPT 的动态加载方式，导致折叠条的开发复杂度大幅提高，稳定性大幅降低。经过综合评估，决定将折叠条功能删除。

关于阅读器，由于当前网页默认只加载视窗内的4~5条消息，以往获取页面内稳定节点的方法已经失效，阅读器无法一次性将所有内容全部展示。为了解决这个问题，针对 ChatGPT 重新设计了内容发现引擎，现在已经具备进入页面之后，阅读器能识别所有消息的能力。

对于官方的浏览页面，经过我的实际使用，我感觉这次更新后的动态加载也有一些不太顺手的地方。比如滚动条快速拉动时，内容显示会有延迟，导致旧消息翻看难度提升。因此，为了照顾喜欢使用原生页面的用户，我顺带将当前的底层能力提取出来做成了目录。进入对话页后，目录将会在右侧展示，支持实时消息预览，并且点击之后能够直接跳转到想要的位置。

## 新增
- 阅读器中的“灵动注释”现在支持“删除注释”（感谢小红书用户 @Z..（研一版））
- 新增 ChatGPT 内右侧目录功能，可以一键跳转到对应对话。

## 变更
- 针对 ChatGPT 新的增量加载机制，重新设计了阅读器的内容发现引擎。
- 调整了 ChatGPT 场景下的书签保存、高亮和跳转策略，使书签与右侧目录共用同一套稳定的对话位置模型。

## 修复
- 修复书签名称无法重命名的问题（感谢小红书用户 @Z..（研一版））
- 修复 ChatGPT 增量加载下书签保存、高亮和跳转定位错误的问题
- 修复 ChatGPT 内阅读器内容获取失败的问题（感谢小红书用户 @沐霖）
- 修复阅读器中发送框内，插入 prompt 功能失效的问题
- 改进书签跳转定位，使 ChatGPT 书签与右侧目录共用同一套稳定锚点链路（感谢小红书用户 @Z..（研一版））。

## 移除
- 删除 ChatGPT 消息折叠功能

# 4.1.0
2026-04-19

这一版的重点有三件事。第一，独立的源码复制面板被删掉了，源码复制能力直接并进了阅读器。第二，新增了“灵动注释”，可以边读边标、边写修改意见，再一次整理成适合发给大模型的输入。第三，书签面板补上了 Changelog、About 和 FAQ 三个信息页，方便查看版本变化、项目背景和常见问题。

其中，把源码复制直接搬到阅读器里，是这次最难做的一部分。公式、代码块、表格、图片这类内容，并不是普通文本，它们更像一个个需要整体处理的闭合单元。如果直接沿用浏览器默认复制，最后拿到的往往只是渲染后的可见文本，原来的 Markdown 结构很容易丢掉。现在的做法，是在阅读器里先识别这些闭合单元对应的源码边界；当你选中它们时，复制逻辑会优先回到原始 Markdown 片段，再按选区顺序重新拼起来，所以复制结果会尽量保持标准 Markdown，而不只是页面上看见的那层文本。

## 新增
- 阅读器里的源码复制现在直接可用，不再需要单独打开源码面板。
- 阅读器支持对代码、公式、表格、图片等闭合内容按源码进行选择和复制，复制结果会尽量保持标准 Markdown 形式。
- 阅读器现在支持对选中内容直接添加灵动注释。选中正文后，会在附近出现悬浮的复制和注释按钮。
- 注释会和对应选区建立关联，并在阅读器中以高亮方式持续标记出来；已有注释也支持再次打开查看和编辑。
- 新增注释导出模板，可在“设置 > 阅读器 > 注释复制模板”中自定义每条注释的导出格式。
- 新增 Prompt Library，可在“设置 > 阅读器 > 用户提示词”中维护多条可复用的自定义提示词。
- 注释结果支持一键复制，也可以直接插入阅读器左下角的发送消息框上方，减少手动复制粘贴。
- 书签面板新增 Changelog、About 和 FAQ 信息页，帮助你更快了解版本变化和项目背景。

## 变更
- 将源码复制能力直接并进阅读器，不再保留独立源码面板作为主要入口。
- 优化 unsupported 页面下的插件 popup，去掉冗余内容，整体样式也更清晰。

## 移除
- 删除独立的源码复制面板。
