# 关于我

AI-MarkDone 最开始不是一个宏大的想法，更像是我给自己补的一块短板。我是一名研究生，平时高频使用 ChatGPT。它确实帮了我很多，但当它慢慢变成我每天都在用的工作台之后，网页里那些不顺手的地方也越来越明显了。

## 为什么我会做 AI-MarkDone

说到底，我会做这个插件，主要就是因为下面这三件事一直在打断我的工作流。

1. 公式相关的问题太多了。前些时候，ChatGPT 的公式经常渲染失败；就算页面上看起来勉强正常，复制出来的 Markdown 也不总是稳定，贴进 Typora 以后还得自己再修一遍。有时候我明明只想拿走其中一个公式，却得先把整段内容复制出来，再从里面把那条公式抠出来，这一步真的很笨。

2. 长对话的体验不够好。对我这种工科生来说，一个对话里出现很多公式是很正常的事，但内容一长，页面就容易变卡，读起来费劲，找东西也慢。工具本来应该帮人省时间，结果反而开始拖慢节奏，这件事我一直很介意。

3. 该有的整理能力没有跟上。分享内容时，它很多时候只能给一个链接，可链接发给别人并不总是好打开，我最后还是得靠截图一点点发。它也没有批量导出的能力，也没有官方收藏功能。再加上我平时经常会做文本润色，明明只想改其中几句，如果不能细到句子做批注，它就很容易把本来没问题的部分也一起改掉。

所以 AI-MarkDone 一开始其实很简单，就是为了解决我自己的这些小麻烦。我不是想把页面改造成另一个产品，而是想把阅读、复制、导出、收藏、批注这些高频动作变得顺手一点。

后来慢慢发现，原来不只是我一个人有这些需求。有人在乎公式，有人在乎导出，有人在乎收藏和整理。每次看到别人说“这个功能正好用上了”，我都会很开心。这也让我更愿意继续把它认真做下去。

如果你愿意一直用它，或者愿意给我提意见，我都很感谢。这个插件到现在还是带着很强的个人起点，但也正因为如此，我会很认真地看每一条反馈。
